Composite with Property

Note: This class diagram is specified in the master file 'urakawa/architecture/coreModel.xmi'.

  • Presentation
    1. Typo: mChannels instead of mChannel (a presentation has several channels).
    2. What's the difference between createNode() and createCompositeNode() ?

      Suggestion: Add a comment or remove a method.

    3. Add the possibility to get a channel (by its name).

      Suggestion: Add a +getChannel(name:String):Channel method to the Presentation class.

    4. Add the possibility to remove a channel by its name.

      Suggestion: Add a +removeChannel(name:String):Channel method to the Presentation class.

  • Node
    1. Add the possibility to detach a node from the tree.

      Suggestion: add a +detach():void method to the Node class.

    2. Add the possibility to get the presentation containing a node.

      Suggestion: add a +getPresentation():Presentation method to the Node class.

    3. Add the possibility to get the index of a given node.

      Suggestion: add a +indexOf(child:Node):int method to the Node class.

    4. Add the possibility to insert a child at a specified index.

      Suggestion: add a +insertChild(newChild:Node,index:int):void method to the Node class.

    5. Add the possibility to directly replace a child by a given node.

      Suggestion: add a +replaceChild(oldChild:Node,newChild:Node):void method to the Node class.

  • Channel
    1. How to get the media objects of a channel ? see the comment ChannelsProperty[1]
    2. Should a channel be restricted to accept only media objects of a same type or is it the responsability of the application ?
  • PropertyType
  • Property
  • StructureType
    1. What are the types for and what do they imply on the StructureProperty ? Do they represent concrete subclasses of StructureProperty ?
    2. Is this repsonsible for managing the mapping between targeted formats and their instances in the urakawa model ? see also StructureProperty[1] and StructureProperty[2].
  • StructureProperty
    1. Should we specify how targeted input/output formats are represented in the urakawa model, ie specify a set of allowed names for the StructureProperty instances ?
    2. How to deal with structures that does not appear explicitly in the input format (for instance mixed content) ?
    3. Manage attributes either directly via getters/setters for the attribute values or by getting the Attribute object.

      Suggestion: add getters/setters for the Attribute objects.

  • Attribute
    1. Add getters and setters for name, namespace, value.
    2. Should we design subclasses for attributes with a special type (int, double, enumeration, etc) ? It's good to avoid redundant string parsing...

      Suggestion: add an AttributeType class, a getType():AttributeType method to the Attribute class, and subclasses of Attribute with custom and typed value getters.

  • SMILProperty (is it a feature of milestone 1)
    1. The usage of this property should be explained, or removed from milestone 1.

      Suggestion: create an xml example of the Urakawa instance for a SMIL document.

  • ChannelsProperty
    1. A ChannelsProperty stores the information for all the channels and there is only one object per channel for a given Node, this should be reflected by this class.

      Suggestion: Replace mMediaObjects with a map of String-MediaObject, the string keys being the Channel names.

    2. Get all the channels that are used in this ChannelsProperty.

      Suggestion: add a +getChannels():ChannelList method to the ChannelsProperty class.

    3. How the ChannelsProperty cope with the SMILProperty ? Could a node override the content of a Channel if it is already defined ? Are these topics the application's responsability ?

      Suggestions: give small and dedicated examples of correct/incorrect ChannelsProperty usage.

Media Object

This class diagram is specified in the master file 'urakawa/architecture/coreModel.xmi'.

  • MediaObject
    1. How to access the represented physical file ? The application may want this for the first iteration, even if the AssetManager isn't fully designed.
  • SequenceMediaObject
    1. Aren''t sequences described by SMILProperty objects ?

      Suggestion: give examples of usage of this class.

  • TextObject
    1. Is this raw text or can it represent all kind of text content ? For instance, how can it be used in Urakawa instances of a SMIL document which has a RealText text element ? A mere String value may be a problem for rich text content.
  • TimedMedia
    1. This terminology can be dangerous in the SMIL context, where text elements are also considered as timed elements. The SMIL recommandation use the terminology "discrete" and "continous" for the distinction of intrinsically timed-base media types and others.

      Suggestion: use the SMIL terminology "continuous media types" for media that are intrinsically timed-base instead of "timed media types".

  • VideoObject
    1. Precise the specification and the expected result of splitVideo() method, is the first part or second part returned ? What about the corresponding MediaFragment ?
  • AudioObject
    1. Precise the specification and the expected result of splitAudio() method, is the first part or second part returned ? What about the corresponding MediaFragment ?
