urakawa.exception Namespace

This namespace contains the toolkit specific exceptions

Namespace hierarchy

Classes

Class Description
ChannelAlreadyExistsException This exception should be thrown when trying to add a Channel whose name is already used in the list of current channels.
ChannelDoesNotExistException This exception should be thrown when trying to remove a Channel whose name does not exist in the list of current channels.
CheckedException Summary description for CheckedException. Exceptions of this type must be caught.
MediaTypeIsIllegalException This exception should be raised when trying to use a MediaType that is not legal in the current context.
MethodParameterIsEmptyStringException Some methods forbid passing empty String values. This exception should be thrown when empty String values are passed.
MethodParameterIsInvalidException Abstract class to encapsulate errors related to wrong values for method parameters. This class cannot be instanciated and should be sub-classed. The aim is to avoid situations where values that are potentially detrimental to software integrity are silently ignored, or "swallowed".
MethodParameterIsNullException Some methods forbid passing NULL values. This exception should be raised when NULL values are passed.
MethodParameterIsOutOfBoundsException Some methods have parameters of numeric type (float, int, uint, etc.). This exception should be thrown when values are out of allowed bounds.
MethodParameterIsWrongTypeException Some methods have stricter type rules than specified by the method signature. This exception should be raised when such type rules are broken.
NodeAlreadyExistException Exception thrown when a previous matching node exists in a child collection, where nodes are supposed to be exclusive
NodeDoesNotExistException Exception thrown when a node does not exists in a child collection
NodeNotDetachedException Exception thrown when trying to insert a node that is not detached
OperationNotValidException (from the design docs) This exception should be thrown/raised when trying to call an operation (aka class method) on an object that does not allow a specific modification of the state in the current context. ... Wherever a "canDoXXX()" method can be found, the corresponding operation "doXXX()" should use this exception/error to let the user-agent of the API/Toolkit know about the non-permitted operation for which there was an attempt to execute.
PropertyTypeIsIllegalException This exception should be raised when trying to use a PropertyType that is not legal in the current context.
TimeOffsetIsNegativeException This exception should be raised when trying to use a time offset that is not allowed to be negative.
TimeStringRepresentationIsInvalidException This exception should be raised when trying to parse an invalid time string representation
UncheckedException Unchecked exceptions do not require catching and handling
